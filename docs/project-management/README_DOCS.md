## ‚úèÔ∏è Contributing to Documentation

When adding new documentation:

1. **Choose the right folder**
   - Setup ‚Üí `guides/`
   - Design ‚Üí `architecture/`
   - Implementation ‚Üí `services/{service}/` or `features/`
   - Bug fixes ‚Üí `fixes/`

2. **Use consistent naming**
   - Format: `DESCRIPTIVE_TOPIC.md`
   - Example: `MOBILE_RESPONSIVE_FIX.md`

3. **Include sections**
   - Overview/Problem statement
   - Solution/Implementation
   - Key changes
   - Testing/Verification

4. **Update INDEX.md**
   - Add entry to appropriate category
   - Include in quick navigation

5. **Link appropriately**
   - Link to related docs
   - Reference parent index
